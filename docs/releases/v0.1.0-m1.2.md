# Release Notes - v0.1.0-m1.2

Release date: 2026-05-05

Tag target: `main` after M1.2 RAG indexing pipeline

## Summary

`v0.1.0-m1.2` marks the completion of the M1.2 RAG indexing sub-milestone.
The backend now has a minimal offline-to-query RAG path:

```text
local paper file -> ingest -> chunker -> BGE embedder -> ChromaStore -> Retriever
```

This is a pre-release milestone for M1. The public chat route is not wired to
RAG yet; that remains M1.3.

## Included

- Added `Embedder` abstraction and lazy-loaded `BGEEmbedder` for `BAAI/bge-large-zh-v1.5`.
- Added `ChromaStore`, a thin wrapper around `chromadb.PersistentClient`.
- Added deterministic character-based chunking with stable `chunk_id` and flat metadata.
- Added ingest helpers for `.txt`, `.md`, and `.pdf` via lazy-imported PyMuPDF.
- Added offline indexing script at `backend/scripts/index_papers.py`.
- Added `Retriever` for query embedding, Chroma lookup, score conversion, and metadata pass-through.
- Added repository-safe sample corpus files under `data/papers/`.
- Added `rag` optional dependency group for `sentence-transformers`, `chromadb`, and `pymupdf`.

## Not Included

- Chat API integration with Retriever. Planned for M1.3.
- Reranking, citation expansion, page-level source mapping, OCR, table extraction, or structured PDF parsing.
- Embedding cache or incremental indexing. Re-running the current indexer is idempotent through `ChromaStore.upsert`.
- Production lifecycle loading for BGE. M1.3 should decide whether to pre-load the embedder during app startup.

## Upgrade Notes

Install RAG extras before running real indexing or PDF ingestion:

```bash
cd backend
uv sync --extra rag
```

Minimal indexer example:

```bash
cd backend
uv run python scripts/index_papers.py \
  --paths ../data/papers/sample.txt \
  --persist-dir var/chroma \
  --collection papers \
  --repo-root ..
```

Generated Chroma state belongs under ignored local paths such as
`backend/var/` or `backend/data/chroma/`.
